# 弱黑名单（衰减版）

对弱黑名单用户/群进行概率回复，连续回复次数越多概率越低，在 LLM 调用前拦截节省 Token。

基于 [astrbot_plugin_weakblacklist](https://github.com/0d00-Ciallo-0721/astrbot_plugin_weakblacklist) 重写。

## 与原版的区别

- **指数衰减算法**：替代固定概率，连续回复越多概率越低（`P(n) = base × decay^n`）
- **LLM 前拦截**：`stop_event()` 阻止 LLM 调用，节省 Token
- **冷却机制**：被拦截后进入冷却期，期间直接拦截，结束后重新开始
- **仅处理 @/唤醒消息**：通过 `is_at_or_wake_command` 过滤，不干扰普通群聊

## 功能

- 用户黑名单 / 群黑名单（独立，OR 关系）
- 指数衰减概率模型，默认 1~3 轮后快速衰减
- 拦截冷却期，防止频繁振荡
- 计数器持久化（JSON），重启不丢失

## 配置项

| 配置 | 说明 |
|---|---|
| 黑名单用户列表 | 弱黑名单用户 ID，私聊群聊均生效 |
| 黑名单群聊列表 | 弱黑名单群号，群内所有人生效 |
| 初始回复概率 | 首次回复概率（默认 0.9） |
| 衰减率 | 每轮概率乘以此值（默认 0.55） |
| 拦截冷却时间 | 被拦截后的冷却分钟数（默认 10） |
| 记录拦截日志 | 是否在日志中记录拦截/放行信息 |
